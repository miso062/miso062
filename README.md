## How I Work

- I like defining the structure first, especially when a project needs to stay maintainable over time.
- I care about clear wording, readable documentation, and how the result will be understood by other people.
- I prefer practical decisions that fit the real operating environment, not just the theoretically ideal answer.
- I verify behavior hands-on by checking UI actions, API responses, database mappings, and external service flows.
- I trace issues through user flow and data flow, then document what happened and what should be improved.
- I pay attention to data consistency, traceability, edge cases, and operational risk.
